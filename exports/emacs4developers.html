<p> But, in order to get the latest version, it's <span class="underline">highly recommended</span> that you compile Emacs yourself: <a href="http://ftp.igh.cnrs.fr/pub/gnu/emacs/">http://ftp.igh.cnrs.fr/pub/gnu/emacs/</a> </p> </div> </div>

<div id="outline-container-sec-4" class="outline-2"> <h2 id="sec-4"><span class="section-number-2">4</span> The basics</h2> <div class="outline-text-2" id="text-4"> </div><div id="outline-container-sec-4-1" class="outline-3"> <h3 id="sec-4-1"><span class="section-number-3">4.1</span> Config files</h3> <div class="outline-text-3" id="text-4-1"> <p> In order to customize Emacs, you will have to edit its configuration files. It does not exist when you install the software (or is an empty file) and does nothing until you decide to tweak Emacs. Some people use Emacs for decades without any extra configuration because they like it out of the box. </p>

<p> There are several ways to manage your Emacs configuration: </p>

<ul class="org-ul"> <li>Simply create and edit a <b>~/.emacs</b> file </li> <li>Create a <b>~/.emacs.d/</b> directory with a <b>~/.emacs.d/init.el</b> file </li> </ul>

<p> The first solution is great if you have a small, tidied and exportable configuration file. </p>

<p> The second solution is great if you need several configuration files (keep you configuration parts separated) or you need to install external extensions (you will install them in <b>~/.emacs.d</b> to keep them centralized and exportable). </p> </div> </div>

<div id="outline-container-sec-4-2" class="outline-3"> <h3 id="sec-4-2"><span class="section-number-3">4.2</span> Executing commands</h3> <div class="outline-text-3" id="text-4-2"> <p> The main idea of Emacs is that everything is a command. For example: opening a file with a keybinding calls a command that is executed by the Emacs' core. And all the commands are written in Lisp. </p>

<p> To call a command, you simply have to press <b>Escape</b> and then <b>x</b>. It could be <b>Alt-x</b> for more comfort. The official name is "<i>Meta x</i>" and is written <i>M-x</i> </p>

<p> If you type this key sequence, you will see at the bottom of the editor a little prompt. This part of Emacs is called the <i>mini buffer</i> and is used for every interactive actions. </p>

<p> Let's try something: </p>

<ul class="org-ul"> <li>type <b>M-x</b> </li> <li>Then, in the minibuffer, type <b>version</b> and <b>RET</b> (return) </li> </ul>

<p> Emacs should display the current version number of the software. </p>

<div class="figure"> <p><img src="../images/version.gif" alt="version.gif" /> </p> <p><span class="figure-number">Figure 1:</span> Execute the version command</p> </div>

<p> Congratulations! You just called your first Emacs command. </p> </div> </div>

<div id="outline-container-sec-4-3" class="outline-3"> <h3 id="sec-4-3"><span class="section-number-3">4.3</span> Key bindings</h3> <div class="outline-text-3" id="text-4-3"> <p> Here are the most useful keys in Emacs. You will use them several hundreds (or thousands) times a day. </p>

<p> How it works? </p>

<ul class="org-ul"> <li>The sequence <i>C-something</i> means that you have to press and hold the <i>Control</i> key while hitting the <i>something</i> key. Example: <i>C-a</i> means <i>Control + a</i>. </li> <li>The sequence <i>C-something somethingelse</i> means that you press <i>Control + something</i>, then you release Control to hit <i>somethingelse</i>. </li> <li>The sequence <i>C-something C-somethingelse</i> means that you have to press Control while hitting <i>something</i> and <i>somethingelse</i>. </li> </ul> </div>

<div id="outline-container-sec-4-3-1" class="outline-4"> <h4 id="sec-4-3-1"><span class="section-number-4">4.3.1</span> Manipulate files</h4> <div class="outline-text-4" id="text-4-3-1"> <ul class="org-ul"> <li><b>C-x C-f</b>: open a file, which corresponds to <b>M-x find-file</b> </li> <li><b>C-x C-s</b>: save a file, which corresponds to <b>M-x save-buffer</b> </li> </ul>

<div class="figure"> <p><img src="../images/open-and-save.gif" alt="open-and-save.gif" /> </p> <p><span class="figure-number">Figure 2:</span> Open a file, modify it and save it</p> </div> </div> </div>

<div id="outline-container-sec-4-3-2" class="outline-4"> <h4 id="sec-4-3-2"><span class="section-number-4">4.3.2</span> Manipulate the line</h4> <div class="outline-text-4" id="text-4-3-2"> <ul class="org-ul"> <li><b>C-a</b>: go to the beginning of the current line, which corresponds to <b>M-x begining-of-line</b> </li> <li><b>C-e</b>: go to the end of the current line, which corresponds to <b>M-x end-of-line</b> </li> <li><b>C-k</b>: cut the rest of the line at the cursor position, which corresponds to <b>M-x kill-line</b> </li> <li><b>C-y</b>: paste what you had cut with the command above, which corresponds to <b>M-x yank</b> </li> <li><b>C-l</b>: center the buffer at the cursor position, which corresponds to <b>M-x recenter-top-bottom</b> </li> </ul>

<div id="outline-container-sec-4-3-3" class="outline-4"> <h4 id="sec-4-3-3"><span class="section-number-4">4.3.3</span> Selecting a region</h4> <div class="outline-text-4" id="text-4-3-3"> <p> Regions are selections, in Emacs. </p>

<p> They are composed by two points. A starting one and and an ending one. </p>

<p> To select the starting point, move your cursor to the wanted location and simply press <b>C-Space</b>. In your minibuffer, you should have seen a message like "Mark set". Now, move your cursor to the location you wantyour selection to end. </p>

<p> You have now a selection (or a <i>region</i>) delimited by the place you set the first mark and your cursor. With this region, you can do what ever you want: </p>

<ul class="org-ul"> <li>Copy (<b>M-w</b>) </li> <li>Cut (<b>C-k</b>) </li> <li>Paste (<b>C-y</b>) </li> <li>Replace (<b>M-%</b>) </li> <li>&#x2026; etc </li> </ul>

<div class="figure"> <p><img src="../images/selecting.gif" alt="selecting.gif" /> </p> <p><span class="figure-number">Figure 4:</span> Selecting a region</p> </div> </div> </div>

<div id="outline-container-sec-4-3-4" class="outline-4"> <h4 id="sec-4-3-4"><span class="section-number-4">4.3.4</span> Commenting</h4> <div class="outline-text-4" id="text-4-3-4"> <p> In order to comment a code block, select the region you want to comment and use: </p>

<ul class="org-ul"> <li><b>M-x comment-region</b> in order to comment the block </li> <li><b>M-x uncomment-region</b> in order to uncomment the block </li> </ul>

<p> Of course, Emacs knows, according to the major mode loaded how to comment properly your code. If the major-mode is <i>ruby-mode</i>, it will add a "#" before the line; if the major-mode is <i>lisp-mode</i>, it will add ";;", and so on and so forth. </p>

<div class="figure"> <p><img src="../images/comment.gif" alt="comment.gif" /> </p> <p><span class="figure-number">Figure 5:</span> Commenting</p> </div> </div> </div>

<div id="outline-container-sec-4-3-5" class="outline-4"> <h4 id="sec-4-3-5"><span class="section-number-4">4.3.5</span> Windows</h4> <div class="outline-text-4" id="text-4-3-5"> <p> This might be confusing but a <i>window</i> is a part of a <i>frame</i>. </p>

<p> An <i>Emacs frame</i> is the window that you opened when you launched Emacs. </p>

<p> An <i>Emacs frame</i> can be divided into <i>windows</i> in itself. </p>

<ul class="org-ul"> <li><b>C-x 2</b>: open a new window horizontally, which corresponds to <b>M-x split-window-below</b> </li> <li><b>C-x 3</b>: open a new window vertically, which corresponds to <b>M-x split-window-right</b> </li> <li><b>C-x o</b>: switch to the next window, which corresponds to <b>M-x other-window</b> </li> <li><b>C-x 0</b>: close the current window, which corresponds to <b>M-x delete-window</b> </li> </ul>

<ul class="org-ul"> <li><b>C-x b</b>: switch to an already opened buffer, which corresponds to <b>M-x switch-to-buffer</b> </li> <li><b>C-x C-b</b>: list opened buffer (and jump to the selected one), which corresponds to <b>M-x list-buffers</b> </li> <li><b>C-x k</b>: kill a buffer, which corresponds to <b>M-x kill-buffer</b> </li> </ul>

<div id="outline-container-sec-4-3-8" class="outline-4"> <h4 id="sec-4-3-8"><span class="section-number-4">4.3.8</span> The most useless one</h4> <div class="outline-text-4" id="text-4-3-8"> <ul class="org-ul"> <li><b>C-x C-c</b>: quit emacs (use at your own risks!), which corresponds to <b>M-x save-buffers-kill-terminal</b> </li> </ul> </div> </div> </div>

<div id="outline-container-sec-4-4" class="outline-3"> <h3 id="sec-4-4"><span class="section-number-3">4.4</span> Help!</h3> <div class="outline-text-3" id="text-4-4"> <p> Emacs has a powerful built-in help system for key bindings and internal functionalities. </p>

<ul class="org-ul"> <li><b>C-h f &lt;function-name&gt;</b>: Find the key binding corresponding to &lt;function-name&gt; (ex: C-h f save-buffer) </li> <li><b>C-h k &lt;key-sequence&gt;</b>: Find the function name corresponding to &lt;key-sequence&gt; (ex: C-h k C-x C-s) </li> </ul>

<ul class="org-ul"> <li><b>C-h r</b>: browse the Emacs manual within Emacs </li> </ul>

<div id="outline-container-sec-5" class="outline-2"> <h2 id="sec-5"><span class="section-number-2">5</span> Building your own editor</h2> <div class="outline-text-2" id="text-5"> <p> From this point, we will edit the configuration file. <div id="outline-container-sec-5-1" class="outline-3"> <h3 id="sec-5-1"><span class="section-number-3">5.1</span> First basic configuration</h3> <div class="outline-text-3" id="text-5-1"> <p> When you will have to change or add configuration, simply edit your <b>~/.emacs.d/init.el</b> file and add what you need. </p>

<div id="outline-container-sec-5-2" class="outline-3"> <h3 id="sec-5-2"><span class="section-number-3">5.2</span> General basic configuration</h3> <div class="outline-text-3" id="text-5-2"> <p> Now, let's move to a more visual configuration basic set. <div class="org-src-container">

<pre class="src src-lisp"><span style="color: #a5a5a5;">;; </span><span style="color: #a5a5a5;">Ask "y" or "n" instead of "yes" or "no". Yes, laziness is great.</span> (fset 'yes-or-no-p 'y-or-n-p)

<div id="outline-container-sec-5-3" class="outline-3"> <h3 id="sec-5-3"><span class="section-number-3">5.3</span> Backup files</h3> <div class="outline-text-3" id="text-5-3"> <p> You may have notice that the files you edit are duplicated and renamed with a <i>~</i> at the end. <div id="outline-container-sec-5-6" class="outline-3"> <h3 id="sec-5-6"><span class="section-number-3">5.6</span> Modes</h3> <div class="outline-text-3" id="text-5-6"> <p> <i>Modes</i> are Emacs' <i>extensions</i> that can be installed to extend the capabilities of Emacs. They will allow you to build a powerful tailored editor. </p>

<p> There are 2 kind of modes: minor and major. </p> </div>

<div id="outline-container-sec-5-6-1" class="outline-4"> <h4 id="sec-5-6-1"><span class="section-number-4">5.6.1</span> Major modes</h4> <div class="outline-text-4" id="text-5-6-1"> <p> Major modes are modes that transform Emacs to a specialized software for editing a certain type of files (i.e c-mode) or managing special tasks (i.e reading emails, managing git repository, &#x2026;) </p>

<p> Only one major mode can be used at a time. </p> </div> </div>

<div id="outline-container-sec-5-6-2" class="outline-4"> <h4 id="sec-5-6-2"><span class="section-number-4">5.6.2</span> Minor modes</h4> <div class="outline-text-4" id="text-5-6-2"> <p> Minor modes are additionnal modes that are added transparently to the major mode. They add more features to the main one (i.e parentheses matching, syntax or spelling checkers, &#x2026;) </p>

<p> Several minor modes can be used at a time. </p> </div> </div> </div>

<div id="outline-container-sec-5-7" class="outline-3"> <h3 id="sec-5-7"><span class="section-number-3">5.7</span> Package managers</h3> <div class="outline-text-3" id="text-5-7"> <p> Emacs has brilliant package managers such as <i>package.el</i> or <i>el-get</i> that allows you to add and update modes really easily. </p> </div>

<div id="outline-container-sec-5-7-1" class="outline-4"> <h4 id="sec-5-7-1"><span class="section-number-4">5.7.1</span> Package.el</h4> <div class="outline-text-4" id="text-5-7-1"> <p> <i>package.el</i> is the built in package manager shipped by default with Emacs 24 or later. </p>

<p> To list available packages, simply type this command: </p>

<div class="org-src-container">

<pre class="src src-lisp">M-x package-list-packages </pre> </div>

<p> You will have a list of packages. Simply press ENTER on the name of one of it to install it. </p>

<p> Additionnaly, you can manage the packages list by adding other sources to your configuration file: </p>

<div class="org-src-container">

<pre class="src src-lisp"><span style="color: #a5a5a5;">;; </span><span style="color: #a5a5a5;">Add package sources</span> (setq package-archives '((<span style="color: #aa3939;">"gnu"</span> . <span style="color: #aa3939;">"http://elpa.gnu.org/packages/"</span>) (<span style="color: #aa3939;">"marmalade"</span> . <span style="color: #aa3939;">"http://marmalade-repo.org/packages/"</span>) (<span style="color: #aa3939;">"melpa"</span> . <span style="color: #aa3939;">"http://melpa.milkbox.net/packages/"</span>))) </pre> </div> </div> </div>

<div id="outline-container-sec-5-7-2" class="outline-4"> <h4 id="sec-5-7-2"><span class="section-number-4">5.7.2</span> El-Get</h4> <div class="outline-text-4" id="text-5-7-2"> <p> <i><a href="https://github.com/dimitri/el-get">el-get</a></i> is one of the most popular and easy to use package manager. The "<i>apt-get</i>" of Emacs. It is written by the great <a href="https://github.com/dimitri">Dimitri Fontaine</a> and is based on recipe files that simply describe where is located the package and how to get/compile/install it for you. </p>

<p> To use it, simply add this to your configuration file. It will download and set up <i>el-get</i> for you: </p>

<div class="org-src-container">

<pre class="src src-lisp"><span style="color: #a5a5a5;">;; </span><span style="color: #a5a5a5;">Set up el-get</span> (add-to-list 'load-path <span style="color: #aa3939;">"~/.emacs.d/el-get/el-get"</span>) (<span style="color: #5FB7CC;">unless</span> (<span style="color: #5FB7CC;">require</span> '<span style="color: #8dd7e9;">el-get</span> nil 'noerror) (with-current-buffer (url-retrieve-synchronously <span style="color: #aa3939;">"https://raw.github.com/dimitri/el-get/master/el-get-install.el"</span>) (<span style="color: #5FB7CC;">let</span> (el-get-master-branch) (goto-char (point-max)) (eval-print-last-sexp)))) </pre> </div> </div> </div>

<div id="outline-container-sec-5-7-3" class="outline-4"> <h4 id="sec-5-7-3"><span class="section-number-4">5.7.3</span> The generic method</h4> <div class="outline-text-4" id="text-5-7-3"> <p> In order to leave you the choice of the package manager you want to use, here is a function that handles several package managers. For now, it covers <i>package.el</i> and <i>el-get</i>. </p>

<p> It also allows you to automatically install packages you want. </p>

<p> In the following snippet, two sample packages are installed </p>

<ul class="org-ul"> <li><i>color-theme</i> in order to allow us to change colors themes </li> <li><i>autopair</i> in order to close automatically parentheses, brackets and braces when you open it </li> </ul>

<p> Simply add this code to your configuration file: </p>

<div class="org-src-container">

<pre class="src src-lisp"><span style="color: #a5a5a5;">;; </span><span style="color: #a5a5a5;">Set up the package manager of choice. Supports "el-get" and "package.el"</span> (setq pmoc <span style="color: #aa3939;">"el-get"</span>)

<span style="color: #a5a5a5;">;; </span><span style="color: #a5a5a5;">List of all wanted packages</span> (setq wanted-packages '( color-theme autopair ))

<span style="color: #a5a5a5;">;; </span><span style="color: #a5a5a5;">Package manager and packages handler</span> (<span style="color: #5FB7CC;">defun</span> <span style="color: #1D68C4;">install-wanted-packages</span> () <span style="color: #aa3939;">"Install wanted packages according to a specific package manager"</span> (interactive) (<span style="color: #5FB7CC;">cond</span> <span style="color: #a5a5a5;">;; </span><span style="color: #a5a5a5;">package.el</span> ((string= pmoc <span style="color: #aa3939;">"package.el"</span>) (<span style="color: #5FB7CC;">require</span> '<span style="color: #8dd7e9;">package</span>) (add-to-list 'package-archives '(<span style="color: #aa3939;">"gnu"</span> . <span style="color: #aa3939;">"http://elpa.gnu.org/packages/"</span>)) (add-to-list 'package-archives '(<span style="color: #aa3939;">"melpa"</span> . <span style="color: #aa3939;">"http://melpa.milkbox.net/packages/"</span>)) (add-to-list 'package-archives '(<span style="color: #aa3939;">"marmelade"</span> . <span style="color: #aa3939;">"http://marmalade-repo.org/packages/"</span>)) (package-initialize) (<span style="color: #5FB7CC;">let</span> ((need-refresh nil)) (mapc (<span style="color: #5FB7CC;">lambda</span> (package-name) (<span style="color: #5FB7CC;">unless</span> (package-installed-p package-name) (set 'need-refresh t))) wanted-packages) (<span style="color: #5FB7CC;">if</span> need-refresh (package-refresh-contents))) (mapc (<span style="color: #5FB7CC;">lambda</span> (package-name) (<span style="color: #5FB7CC;">unless</span> (package-installed-p package-name) (package-install package-name))) wanted-packages) ) <span style="color: #a5a5a5;">;; </span><span style="color: #a5a5a5;">el-get</span> ((string= pmoc <span style="color: #aa3939;">"el-get"</span>) (add-to-list 'load-path <span style="color: #aa3939;">"~/.emacs.d/el-get/el-get"</span>) (<span style="color: #5FB7CC;">unless</span> (<span style="color: #5FB7CC;">require</span> '<span style="color: #8dd7e9;">el-get</span> nil 'noerror) (with-current-buffer (url-retrieve-synchronously <span style="color: #aa3939;">"https://raw.github.com/dimitri/el-get/master/el-get-install.el"</span>) (<span style="color: #5FB7CC;">let</span> (el-get-master-branch) (goto-char (point-max)) (eval-print-last-sexp)))) (el-get 'sync wanted-packages)) <span style="color: #a5a5a5;">;; </span><span style="color: #a5a5a5;">fallback</span> (t (<span style="color: #ffc0cb; font-weight: bold;">error</span> <span style="color: #aa3939;">"Unsupported package manager"</span>))) )

<span style="color: #a5a5a5;">;; </span><span style="color: #a5a5a5;">Install wanted packages</span> (install-wanted-packages) </pre> </div>

<p> Note that some of the following package names could vary if you use <i>package.el</i> or <i>el-get</i>. </p> </div> </div>

<div id="outline-container-sec-5-7-4" class="outline-4"> <h4 id="sec-5-7-4"><span class="section-number-4">5.7.4</span> The modern and efficient way</h4> <div class="outline-text-4" id="text-5-7-4"> <p> Some upper-level packages make package management easier and cleaner. </p>

<p> For example, John Wiegley's <code>use-package</code> software is an elegant and efficient way to install and tidy your external packages configurations. </p>

<p> Here is the way to use it: </p>

<p> First, setup packages sources and install <code>use-package</code> </p>

<div class="org-src-container">

<div id="outline-container-sec-5-9" class="outline-3"> <h3 id="sec-5-9"><span class="section-number-3">5.9</span> Code navigation</h3> <div class="outline-text-3" id="text-5-9"> <p> Navigate through source code is an indispensable feature in code editors. Some of other editors are focused on this (like Sublime) but the problem is that they do not leave you the choices of the weapons (like every other features). </p>

<p> Emacs, as you noticed (yes, you did) can integrate any external tool or include a large variety of internal tools to make it more efficient. Code navigation is not an exception. </p>

<p> There are a lot of packages in order to navigate through code, with different methodologies and advantages: </p>

<ul class="org-ul"> <li><a href="http://www.emacswiki.org/emacs/EmacsTags">TAGS</a> (built in. No installation required) </li> <li><a href="https://github.com/bbatsov/projectile">Projectile</a> (install it by adding <code>projectile</code> to your packages list) </li> <li><a href="https://github.com/emacs-helm/helm">Helm</a> (install it by adding <code>heml</code> to your packages list) </li> <li><a href="http://cedet.sourceforge.net/">Emacs CEDET</a> (install it by adding <code>cedet</code> to your packages list) </li> <li>&#x2026; </li> </ul>

<p> (Note: use once at a time! You do not need to install them all) </p>

<p> After using <i>Projectile</i> for a certain period of time, I gave <i>Helm</i> a shot but finally got back to the simplicity and the efficiency of the <i>TAGS</i> system. Nevertheless, <i>Projectile</i> and <i>Helm</i> are really powerful and useful. You really should look at them. </p>

<p> For now, I will cover the basics of <i>TAGS</i> since it is a standard (used by other softwares) and it is built-in (so if you use another Emacs without your config, you still know how to surf in the code). And other useful tricks will be shown here. </p> </div>

<div id="outline-container-sec-5-9-1" class="outline-4"> <h4 id="sec-5-9-1"><span class="section-number-4">5.9.1</span> Tags basics</h4> <div class="outline-text-4" id="text-5-9-1"> <p> Basically, the TAGS rely on an index file of the content of your code source. In the root folder of a project, you must generate your index. Several methods for this, but here is one command that generates the file: </p>

<div class="org-src-container">

<pre class="src src-sh"><span style="color: #7FFFD4;">cd</span> /path/to/your/project find . -regex <span style="color: #aa3939;">".*\.\(c\|h\|rb\|py\|php\|js\|sh\|bash\)"</span> -print | xargs etags -a </pre> </div>

<p> You now should see a "TAGS" file. If needed, exclude this file from your git/svn/whatever repository/. </p>

<p> You are now ready to surf! </p>

<p> Now, here are the main commands to use them: </p>

<ul class="org-ul"> <li><b>M-.</b> is the equivalent of <b>M-x find-tag</b>: find a tag </li> <li><b>M-x find-tag-other-window</b>: find a tag, but in another window </li> <li><b>M-x tags-search</b>: find a tag thanks to a regexp </li> <li><b>M-x tags-query-replace</b>: replace a tag in all the indexed files </li> <li><b>M-x tags-apropos</b>: list all tags that match a regexp </li> <li><b>M-x list-tags</b>: list all tags in a file </li> </ul> </div> </div>

<div id="outline-container-sec-5-9-2" class="outline-4"> <h4 id="sec-5-9-2"><span class="section-number-4">5.9.2</span> List functions or methods</h4> <div class="outline-text-4" id="text-5-9-2"> <p> In order to list and jump easily between the functions or methods of the current file, here is a little trick: </p>

<div class="org-src-container">

<pre class="src src-emacs-lisp">(global-set-key (kbd <span style="color: #aa3939;">"C-S-f"</span>) 'imenu) <span style="color: #a5a5a5;">;; </span><span style="color: #a5a5a5;">use iMenu</span> </pre> </div>

<p> After pressing Ctrl-Shift-f (replace it by whatever you want!), a menu with all the available resources appears in the minibuffer. Select the resource you want (let's say <i>All.methods</i> if you are editing some code) and you will be able to search and jump to a method definition directly. </p> </div> </div>

<div id="outline-container-sec-5-9-3" class="outline-4"> <h4 id="sec-5-9-3"><span class="section-number-4">5.9.3</span> Files navigation (a.k.a fuzzy matching) with TAGS and ido</h4> <div class="outline-text-4" id="text-5-9-3"> <p> In order to find and jump easily to a file in your project thanks to TAGS, you have to add a little function to your confiugration. For this you should have installed the <code>ido-mode</code> described above. </p>

<div class="org-src-container">

<pre class="src src-emacs-lisp">(<span style="color: #5FB7CC;">defun</span> <span style="color: #1D68C4;">ido-find-file-in-tag-files</span> () (interactive) (<span style="color: #5FB7CC;">save-excursion</span> (<span style="color: #5FB7CC;">let</span> ((enable-recursive-minibuffers t)) (visit-tags-table-buffer)) (find-file (expand-file-name (ido-completing-read <span style="color: #aa3939;">"Project file: "</span> (tags-table-files) nil t)))))

(global-set-key (kbd <span style="color: #aa3939;">"C-S-x C-S-f"</span>) 'ido-find-file-in-tag-files) </pre> </div>

<p> Now, in your project folder and once you generated your TAGS file, you can press <b>C-S-x C-S-f</b> to find files through the whole tree of directories just by typing a pattern. </p>

<div class="figure"> <p><img src="../images/fuzzy.gif" alt="fuzzy.gif" /> </p> </div> </div> </div>

<div id="outline-container-sec-5-9-4" class="outline-4"> <h4 id="sec-5-9-4"><span class="section-number-4">5.9.4</span> RGrep</h4> <div class="outline-text-4" id="text-5-9-4"> <p> <i>rgrep</i> is a great tool to use in addition of the TAGS. It is a Unix tool that make <i>grep</i> queries recursively. In Emacs, it will bring you a new buffer with all the results of the command. To use it, simply type: </p>

<p> <b>M-x rgrep RET yoursearchterm RET</b> </p>

<p> Once fired, you can swith to the newly created buffer, parse the results and jump to the wanted files very quickly. </p>

<p> What is cool is that it is integrated in the editor, just beside your code and does not require to switch to a shell and then copy-paste the files path you want. </p>

<div id="outline-container-sec-7" class="outline-2"> <h2 id="sec-7"><span class="section-number-2">7</span> Special features</h2> <div class="outline-text-2" id="text-7"> </div><div id="outline-container-sec-7-1" class="outline-3"> <h3 id="sec-7-1"><span class="section-number-3">7.1</span> Emacs as a file manager</h3> <div class="outline-text-3" id="text-7-1"> <p> Emacs has a built-in mode named <i>dired</i> that allows you to manage your file system directly in Emacs really easily. It is very powerful and has features that graphical file managers do not have. </p>

<p> First of all, to launch it, type: </p>

<ul class="org-ul"> <li><b>M-x dired RET</b> and then select a path to open (default is the directory of the file you are editing) </li> <li>or <b>C-x d</b> if you prefer keybindings </li> </ul>

<p> Here are a few quick shortcuts once you are in <i>dired mode</i> </p>

<ul class="org-ul"> <li><b>R</b> (capital R): rename a file </li> <li><b>D</b> (capital D): delete a file </li> <li><b>+</b>: create a new directory </li> <li><b>Z</b>: compress the file </li> <li><b>RET</b> (enter): Open the file </li> <li><b>g</b>: refresh </li> <li><b>q</b>: close the dired window </li> </ul>

<p> Of course, files can be marked to operate on a selection of them. Use <b>m</b> to mark, <b>u</b> to unmark (<b>U</b> to unmark all), <b>% m</b> to mark according to a pattern. </p>

<p> Note that there are so many tricks, extensions and features in <i>dired mode</i> that I let you see around what you need. </p>

<p> <a href="http://www.emacswiki.org/emacs/DiredMode">Read more about Dired</a> </p>

<p> <a href="https://www.gnu.org/software/emacs/manual/html_node/emacs/Dired.html">Read the manual</a> </p> </div> </div>

<div id="outline-container-sec-7-2" class="outline-3"> <h3 id="sec-7-2"><span class="section-number-3">7.2</span> Working locally vs remotely</h3> <div class="outline-text-3" id="text-7-2"> <p> Emacs, once installed on your machine will allow you to edit your file locally, obviously. </p>

<p> Other people (non-Emacs users) will do a dirty trick to edit remote files like they were local files. They will mount the remote directory on their machine (thanks to <i>sshfs</i>). But the magic in Emacs is that you can transparently edit remote files on your development servers for example. To do so, Emacs comes with a genious extension named "Tramp". It is alreay installed and available when you install Emacs. </p>

<p> If you press <b>C-x C-f</b>, you can open a file. But if you ask <code>myname@myserver.com:/path/to/file</code>, it will automatically connect to the server and let you edit the remote file. Easy as pie. <i>Tramp</i> supports lots of protocols like ssh, ftp, and so on. </p>

<p> And what is great is that if you are editing a remote file and you launch a shell in Emacs, it will automatically set the shell into your remote environment and open it as if you were in the remote directory the file is in. </p>

<p> A last tip about TRAMP. If you simply add <code>sudo:</code> in front of your file path, you edit your file with higher privileges. It is a very useful feature if you need to edit your configuration files on a server, for example. Of course, it works remotely and locally. </p>

<p> <a href="http://www.emacswiki.org/emacs/TrampMode">Read more about Tramp</a> </p> </div> </div>

<div id="outline-container-sec-7-4" class="outline-3"> <h3 id="sec-7-4"><span class="section-number-3">7.4</span> Emails in Emacs</h3> <div class="outline-text-3" id="text-7-4"> <p> There are packages that let you use Emacs as a full featured MUA. Reading emails, sending emails, filtering, archiving &#x2026; etc. </p>

<p> Emacs can be a very powerful and fast e-mail client and as it is controlled by the keyboard, it could be more efficient than a graphical e-mail client (clicking is a loss of time). I use it for years now and I can not imagine using another program to read my emails. </p>

<p> There are several package to do so, but the most used is <a href="http://www.gnus.org/">Gnus</a>. It is already installed with Emacs. </p>

<p> Here are some of its features beside the basic ones: </p>

<ul class="org-ul"> <li>Gnus is in fact a newsreader used to interact with email servers. So you can use to fetch messages from newsgroups, RSS, SMTP, POP, &#x2026; </li> <li>Easy and powerful mail splitting </li> <li>Integration with BBDB, a contact manager </li> <li>Message scoring </li> <li>LDAP </li> <li>PGP signing and encrypting </li> <li>Customizable layout </li> <li>Encrypted file to manage your credentials </li> <li>&#x2026; and many more </li> <li>&#x2026; and of course, it is extensible thanks to Lisp </li> </ul>

<p> You really should try it with your personnal account. It is easy to configure and integrate with <a href="http://www.emacswiki.org/emacs/GnusGmail">Gmail</a>, for example. </p>

<p> But be careful! If you try it, you will love it. </p>

<p> Some alternatives to Gnus (that I do not use, but that are popular): </p> <ul class="org-ul"> <li><a href="http://www.emacswiki.org/emacs/WanderLust">Wanderlust</a> </li> <li><a href="http://www.djcbsoftware.nl/code/mu/mu4e.html">Mu4e</a> </li> </ul> </div> </div>
